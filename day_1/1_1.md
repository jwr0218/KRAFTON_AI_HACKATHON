# [보고서] Problem 1-1: Exact Binary Multiplication via Zero-Parameter Routing & Algebraic Carry

## 1. 구현 알고리즘 (Implemented Algorithm)

본 트랜스포머 모델이 구현하는 알고리즘은 **LSB-First 이진 부분곱 합산(Shift-and-add Binary Multiplication)** 및 **대수학적 자리올림(Algebraic Carry Deduction)** 알고리즘입니다.

일반적인 가중치 학습 패러다임을 우회하여, 트랜스포머의 Attention 메커니즘을 **조건부 물리적 라우팅(Routing)** 으로, MLP를 **디지털 논리 게이트(AND, Modulo 2)** 로 치환하여 설계했습니다.

---

## 2. 레이어별 정확성 증명 (Layer-by-Layer Correctness Proof)

### Phase 1 (Layer 1): 부분곱 계산 및 라우팅 (Partial Sum & AND Gate)

Layer 1의 목적은 입력된 피연산자 $A$와 $B$로부터 출력될 $k$번째 자리수에 기여하는 비트 쌍들을 매칭하고, 논리곱(AND)을 수행하여 해당 자리의 순수 합계($\text{Partial\_Sum}_k$)를 구하는 것입니다.

#### Attention (What it attends to)

특정 출력 비트 $P_k$를 생성하기 위해 쿼리(Query)가 발생하면, $i + j = k$를 만족하는 입력 $A_i$와 $B_j$ 쌍을 찾아야 합니다.

본 아키텍처는 **6개의 독립적인 어텐션 헤드($c \in \{0, 1, ..., 5\}$)** 를 사용합니다.

헤드 $c$는 고정된 마스크 텐서를 통해 항상 $A_c$(인덱스 $c$)와 $B_{k-c}$(인덱스 $6+k-c$) 토큰에만 정확히 $1.0$의 Attention 점수를 부여합니다.

조건($0 \le k-c \le 5$)을 벗어나는 잉여 쿼리는 안전한 패딩(Padding) 인덱스로 라우팅하여 프레임워크의 연산 오류(NaN)를 방지하는 **0-Parameter 라우팅** 을 달성했습니다.

#### MLP (What it computes)

각 헤드에서 수집된 두 비트의 스칼라 합($x \in \{0, 1, 2\}$)이 개별적으로 MLP를 통과합니다.

MLP는 다음과 같은 비선형 임계값 함수를 계산합니다.

$$
f(x) = \text{ReLU}(2x - 1.5)
$$

이 식은 입력된 두 비트가 모두 $1$일 때(즉, 스칼라 합이 2일 때)만 양수 값을 반환하므로 완벽한 AND 게이트를 구성합니다.

6개 헤드의 출력을 합산하여 $P_k$ 자리에 위치해야 할 1의 총개수, 즉 $\text{Partial\_Sum}_k$를 산출합니다.

---

### Phase 2 (Layer 2): 대수학적 자리올림 및 비트 추출 (Carry Deduction & Modulo 2)

트랜스포머는 자기회귀(Autoregressive) 생성 구조상 이전 스텝의 은닉 상태(Hidden State)를 직접 넘겨받지 못합니다.

Layer 2는 과거에 생성된 출력 토큰들을 참조하여 자리올림(Carry)을 수학적으로 역산하고, 최종 출력 비트를 결정합니다.

#### Attention (What it attends to)

인과적 마스킹(Causal Mask)이 적용된 상태에서, $P_k$ 위치의 어텐션은 **과거의 모든 $\text{Partial\_Sum}_m$ 및 실제 생성된 출력 비트 $P_m$ ($m < k$)** 에 주의를 기울입니다.

어텐션 가중치는 현재 자리와의 거리 차이에 따라 지수적으로 감쇠하는 $2^{-(k-m)}$의 스케일을 갖도록 하드코딩되었습니다.

이를 통해 다음의 대수학적 역산 공식이 성립하며, 이전 스텝들로부터 누적되어 넘어온 자리올림수 $C_k$를 무손실로 복원해 냅니다.

$$
C_k = \sum_{m=0}^{k-1} (\text{Partial\_Sum}_m - P_m) \cdot 2^{-(k-m)}
$$

#### MLP (What it computes)

Layer 1의 결과와 Layer 2 어텐션의 결과를 더해 현재 자리의 총합($\text{Total}_k = \text{Partial\_Sum}_k + C_k$)을 구합니다.

이후 MLP는 교대합(Alternating Sum) 원리를 활용하여 다음과 같은 비선형 결합을 연산합니다.

$$
f(x) = \text{ReLU}(x) - 2\text{ReLU}(x-1) + 2\text{ReLU}(x-2) - 2\text{ReLU}(x-3) \dots
$$

이 연산은 $\text{Total}_k \pmod 2$를 완벽하게 모사하는 톱니파(Pulse) 발생기 역할을 합니다.

즉, 총합이 홀수일 때만 1을 뱉어내는 **짝/홀수 판별기(Parity Checker)** 로 기능합니다.

---

## 3. 최종 출력의 구성 (Formation of the Output)

최종 Output Head는 Layer 2의 펄스 발생기를 통과한 스칼라 값($0$ 또는 $1$)을 받아 토큰 확률 로짓(Logit)으로 변환합니다.

$P_k$는 철저히 $\pmod 2$ 연산을 통해 LSB(Least Significant Bit)부터 한 자리씩 생성되며, 이렇게 결정된 비트가 다음 스텝의 Layer 2 어텐션에 참조되어 $\text{Total} - \text{Mod}(\text{Total})$이라는 완벽한 짝수 뺄셈을 유도합니다.

결과적으로 트랜스포머는 12번의 자기회귀 스텝을 거치며 논리적으로 무결한 12비트 곱셈 결과를 차례대로 구성하게 됩니다.
